# P3D-007 — Tree, TreeNode, TreeNode2 (structure spatiale de culling)

- **Status:** TODO
- **Priority:** P2
- **Module:** P3D
- **Depends on:** —
- **Files:** `dev/codegen/p3d.json:950-963` (`Tree`, `TreeNode` **déjà décrits**), `src/P3D/P3DChunk.h:163-165` (ChunkType), `src/P3D/P3D.generated.{h,cpp}`, `src/Render/`
- **Blocks:** RENDER-008b

## Contexte

Arbre spatial (BVH/octree-like) utilisé pour le frustum culling. État inégal :

- `Tree` (0x3F00004) : `version u32`, `name string`, `numNodes u32`,
  `nodes children TreeNode` — **déjà dans `p3d.json`** mais non régénéré ;
- `TreeNode` (0x3F00005) : `version u32`, `min vec3`, `max vec3`, `numChildren u32`,
  `children children TreeNode` — **déjà dans `p3d.json`** (récursif) mais non régénéré ;
- `TreeNode2` (0x3F00006) : déclaré dans l'enum `ChunkType` mais **absent de `p3d.json`**
  → à reverser (variante, probablement avec indices de drawables aux feuilles).

## Approche

1. Lancer `dev/codegen/run.bat` pour générer `Tree` et `TreeNode` (récursif via
   `children TreeNode`, motif déjà supporté — cf. `SceneGraphTransform`).
2. Reverser `TreeNode2` : confirmer s'il porte les références aux objets de la feuille
   (indices d'entités à dessiner quand le nœud est visible). L'ajouter à `p3d.json`,
   régénérer.
3. Brancher côté `Render/` : utiliser les bounding boxes (`min`/`max`) des nœuds pour
   réaliser le frustum culling hiérarchique (base de **RENDER-008b**) — descendre l'arbre,
   rejeter les sous-arbres hors frustum.

## Critères d'acceptation

- [ ] `Tree`, `TreeNode` générés ; structure récursive correctement parsée.
- [ ] `TreeNode2` reversé, décrit dans `p3d.json` et généré.
- [ ] Un `.p3d` contenant un `Tree` se parse sans assert (bounds cohérents).
- [ ] Les bounding boxes des nœuds sont exposées pour le culling (consommables par RENDER-008b).
