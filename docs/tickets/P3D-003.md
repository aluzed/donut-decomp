# P3D-003 — Chunks d'expression / animation faciale

- **Status:** TODO
- **Priority:** P3
- **Module:** P3D
- **Depends on:** —
- **Files:** `src/P3D/P3DChunk.h:104-106,54` (ChunkType), `dev/codegen/p3d.json` (à compléter), `src/P3D/P3D.generated.{h,cpp}`

## Contexte

Chunks d'expression faciale (morph targets), déclarés dans l'enum `ChunkType` mais
**absents de `p3d.json`** (donc non parsés) :

- `Expression` (0x21000), `ExpressionGroup` (0x21001), `ExpressionMixer` (0x21002).

À noter : `ExpressionOffsets` (0x10018) **est déjà décrit dans `p3d.json`**
(`size u32`, `offsets buffer<vec3[size]>`) mais **pas encore régénéré** — c'est la liste de
décalages de vertices utilisée par les expressions. Basse priorité (cosmétique facial).

## Approche

1. Reverser les layouts de `Expression`, `ExpressionGroup`, `ExpressionMixer` (Ghidra /
   dumps de modèles de personnages). Hiérarchie probable : `ExpressionGroup` agrège des
   `Expression`, chaque `Expression` référençant un `ExpressionOffsets` ; `ExpressionMixer`
   pondère plusieurs expressions.
2. Les décrire dans `p3d.json` ; régénérer `ExpressionOffsets` au passage via
   `dev/codegen/run.bat`.
3. Vérifier les classes et `Load()`.
4. (Optionnel, hors scope parsing) brancher au mélange de morph targets côté `Render/`.

## Critères d'acceptation

- [ ] `Expression`, `ExpressionGroup`, `ExpressionMixer` décrits dans `p3d.json` et générés.
- [ ] `ExpressionOffsets` régénéré et accessible.
- [ ] Un `.p3d` de personnage avec expressions se parse sans assert.
- [ ] Les offsets de morph et leur association aux expressions sont exposés via getters.
