# FX-004 — Shadow mapping (FBO de profondeur directionnel)

- **Status:** TODO
- **Priority:** P3
- **Module:** Post-FX
- **Depends on:** FX-000
- **Files:** `src/Render/OpenGL/FrameBuffer.h:27-31,:83`, `src/Game.cpp:833-857`, `assets/shaders/world.vert`, `assets/shaders/world.frag`
- **Note:** anciennement RENDER-012

## Contexte
Aucune ombre projetée : le rendu monde (`Game.cpp:845-846`) et le personnage
(`:854`) n'ont pas de passe de profondeur depuis la lumière. Le `world.frag`
applique seulement diffuse + lightmap + fog. Le `GL::FrameBuffer` supporte déjà un
depth buffer en texture (`FrameBuffer.h:27-31`, `BindDepthTexture` `:83`),
infrastructure suffisante pour une shadow map directionnelle.

## Approche
1. **Passe ombre** : créer un FBO depth-only (`Format::EnableColourBuffer(false)`,
   `EnableDepthBuffer(true, true)`, `FrameBuffer.h:27-31`). Calculer une matrice
   `lightViewProj` orthographique alignée sur la direction du soleil (cf. lumière
   du `WorldSphere`).
2. Rendre la géométrie occludante depuis la lumière avec un shader depth minimal
   (réutiliser `world.vert` sans frag couleur), avant la passe scène
   (`Game.cpp:833`).
3. **Passe scène** : passer `lightViewProj` + la depth texture (unité dédiée) à
   `world.vert`/`world.frag` ; calculer la coord ombre, comparer la profondeur,
   appliquer un facteur d'ombrage (PCF simple) avec bias anti acné.
4. Intégrer proprement avec FX-000 : la passe scène écrit toujours dans
   `_sceneFBO` ; la shadow map est une passe séparée en amont. Exposer
   résolution / intensité via ImGui.

## Critères d'acceptation
- [ ] Les objets projettent des ombres directionnelles cohérentes avec le soleil.
- [ ] Pas d'acné d'ombre marquée (bias/PCF) ni de peter-panning excessif.
- [ ] La shadow map se régénère par frame ; activable/désactivable sans crash.
