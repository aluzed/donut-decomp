## Contexte

Cinq chunks de géométrie **déjà entièrement décrits dans `p3d.json`** mais **non
régénérés** (absents de `P3D.generated.h`) :

- `Mesh` (similaire à `Geometry`/`PolySkin` : `numPrimGroups` + `children PrimitiveGroup`
  + `boundingBox` + `boundingSphere`) ;
- `OffsetList` (0x1000E) : `size u32`, `offsets buffer<vec3[size]>` ;
- `PackedNormalList` (0x10010) : `size u32`, `normals buffer<u32[size]>` (normales
  compressées) ;
- `VertexShader` (0x10011) : `version u32`, `data buffer<u8[dataSize]>` ;
- `DrawShadow` (0x10017) : `version u32`, `todo u32`.

## Approche

1. Vérifier les layouts (notamment `VertexShader` : `dataSize` n'est pas un champ déclaré —
   confirmer que le générateur résout `buffer<u8[dataSize]>` via la taille du chunk ;
   sinon corriger en `buffer<u8>`).
2. Confirmer le format des normales packées de `PackedNormalList` (décodage `u32` →
   `vec3`) côté consommateur.
3. Lancer `dev/codegen/run.bat`, vérifier les 5 classes et leurs `Load()`.
4. Brancher dans `ResourceManager`/`CompositeModel` : les `PrimitiveGroup` d'un `Mesh`
   suivent le même chemin que `Geometry` (`rm.LoadGeometry`) ; `OffsetList` /
   `PackedNormalList` alimentent les attributs de vertex.

## Critères d'acceptation

- [ ] Les 5 classes existent dans `P3D.generated.h` après `run.bat`.
- [ ] `VertexShader` lit son blob de données sans dépassement de `MemoryStream`.
- [ ] Un `.p3d` contenant un `Mesh` se charge et s'affiche (même chemin que `Geometry`).
- [ ] `PackedNormalList` est décodable en normales `vec3` exploitables.
