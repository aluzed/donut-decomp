# FX-005 — Color grading par LUT (texture 16³)

- **Status:** TODO
- **Priority:** P3
- **Module:** Post-FX
- **Depends on:** FX-000
- **Files:** `assets/shaders/postprocess.frag:15-21,:58`, `src/Render/OpenGL/GLTexture2D.h`, `src/Game.cpp:183-185`

## Contexte
Le color grading actuel du `postprocess.frag` est purement analytique
(saturation + contraste, `postprocess.frag:15-21`, appelé `:58`). Une LUT 3D
(16×16×16) permettrait des étalonnages artistiques arbitraires. Nécessite la
passe post-process (FX-000) et une texture 3D (ou 2D « unrolled » 256×16).

## Approche
1. Charger une LUT : soit `GL_TEXTURE_3D` 16³ RGB, soit une LUT 2D dépliée
   (256×16) si la chaîne `GLTexture2D` est plus simple à étendre
   (`GLTexture2D.h`). Filtrage linéaire, clamp aux bords.
2. Dans `postprocess.frag`, ajouter `uniform sampler3D colorLUT;` (ou sampler2D
   pour la version dépliée) et remplacer/compléter `applyColorGrading`
   (`:15-21`) par un échantillonnage LUT : `color = texture(colorLUT, color)`
   (avec offset demi-texel pour éviter le filtrage hors-cellule).
3. Côté moteur (après FX-000), créer/charger la texture LUT, la binder sur une
   unité dédiée, fournir l'uniform sampler. Prévoir une LUT « identité » par
   défaut (passthrough).
4. Slider/sélecteur de LUT dans l'overlay debug ImGui (`Game.cpp` ~l.1277).

## Critères d'acceptation
- [ ] Une LUT identité ne modifie pas l'image (égalité avec passthrough).
- [ ] Changer de LUT modifie visiblement l'étalonnage couleur.
- [ ] Pas de banding marqué ni de débordement de cellule (offset demi-texel +
      filtrage linéaire).
