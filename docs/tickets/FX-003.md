# FX-003 — Motion blur (reprojection du viewProj précédent)

- **Status:** TODO
- **Priority:** P3
- **Module:** Post-FX
- **Depends on:** FX-000
- **Files:** `src/Game.cpp:841-843`, `assets/shaders/postprocess.frag`, `src/Render/OpenGL/FrameBuffer.h`

## Contexte
Pas de flou de mouvement. La caméra recalcule `viewProjection` chaque frame
(`Game.cpp:841-843`) mais ne conserve pas la matrice précédente. Un motion blur
camera-based reprojette chaque pixel via `prevViewProj * inverse(viewProj)` pour
estimer un vecteur de mouvement écran et accumuler le long de celui-ci. Nécessite
la passe post-process (FX-000) et l'accès à la profondeur de scène.

## Approche
1. Stocker `_prevViewProj` dans `Game` et le mettre à jour en fin de frame (après
   `Game.cpp:843`). Conserver aussi l'inverse du `viewProj` courant.
2. S'assurer que `_sceneFBO` (FX-000) expose une **depth texture**
   (`Format::EnableDepthBuffer(true, true)`, `FrameBuffer.h:27-31`) pour
   reconstruire la position monde de chaque pixel.
3. Dans `postprocess.frag` : reconstruire la position monde depuis la profondeur,
   la reprojeter avec `prevViewProj` pour obtenir l'UV précédent, dériver le
   vecteur de vélocité écran, puis échantillonner N pas de `sceneTex` le long de
   ce vecteur (clamp de longueur pour éviter le smearing).
4. Exposer l'intensité / nombre d'échantillons via uniforms + slider ImGui
   (`Game.cpp` ~l.1277). Intensité 0 ⇒ passthrough.

## Critères d'acceptation
- [ ] Un mouvement rapide de caméra produit un flou directionnel cohérent avec le
      déplacement.
- [ ] Intensité 0 (ou caméra immobile) ⇒ image identique au passthrough.
- [ ] Pas de smearing excessif sur les bords ni d'artefacts aux discontinuités de
      profondeur (longueur de vélocité bornée).
