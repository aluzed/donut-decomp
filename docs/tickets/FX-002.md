# FX-002 — Vignette (bords assombris, configurable)

- **Status:** TODO
- **Priority:** P3
- **Module:** Post-FX
- **Depends on:** FX-000
- **Files:** `assets/shaders/postprocess.frag:8,:52-54,:60`, `src/Game.cpp:183-185`

## Contexte
Le `postprocess.frag` contient déjà la vignette : `vignetteIntensity`
(`postprocess.frag:8`), le `smoothstep` radial corrigé du ratio d'aspect
(`:52-54`) et la multiplication finale `color *= vignette` (`:60`). Il manque
seulement le pilotage de l'uniform depuis le moteur, une fois la passe
post-process branchée (FX-000).

## Approche
1. Une fois FX-000 en place, exposer `vignetteIntensity` (et éventuellement les
   bornes du `smoothstep`) comme paramètre runtime du `_postProcessShader`.
2. Dans la passe finale (cf. FX-000), `_postProcessShader->SetUniformValue(
   "vignetteIntensity", value)` ; valeur par défaut raisonnable (~0.3) et 0 =
   désactivé.
3. Ajouter un slider dans l'overlay debug ImGui (`Game.cpp` ~l.1277) pour régler
   l'intensité à chaud.
4. Vérifier la correction d'aspect : `screenSize.x/screenSize.y` est déjà utilisé
   (`postprocess.frag:54`) — s'assurer que `screenSize` est bien fourni par FX-000.

## Critères d'acceptation
- [ ] Les bords de l'image s'assombrissent progressivement, centre intact.
- [ ] `vignetteIntensity = 0` redonne une image identique au passthrough.
- [ ] La vignette reste circulaire/cohérente en 16:9 comme en 4:3 (aspect
      corrigé).
