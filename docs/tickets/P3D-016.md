# P3D-016 — Reverse de la compression P3DZ (header 0x5A443350)

- **Status:** TODO
- **Priority:** P2
- **Module:** P3D
- **Depends on:** —
- **Files:** `src/P3D/P3DFile.cpp:12-59`, `src/P3D/P3DFile.h:14-19` (enum `FileTypes`), `src/P3D/P3DChunk.h:197` (`P3DZRoot = 0x5A443350`)

## Contexte

`P3DFile` gère deux formats : `P3D` (0xFF443350, non compressé) et `RZ` (0x5A52, zlib
deflate, décompressé via `uncompress`). Le troisième, **`P3DZ` (0x5A443350, 'P3DZ',
compression propriétaire)**, est déclaré dans `FileTypes` (commenté « proprietary
compression ») et dans `ChunkType::P3DZRoot`, mais **n'est pas géré** : le constructeur
`P3DFile::P3DFile` ne teste que `RZ` puis `assert(type == P3D)` → tout `.p3d` P3DZ fait
échouer l'assert. Format inconnu, doit être reversé.

## Approche

1. Collecter des échantillons `.p3d` commençant par `0x5A443350` (LE → octets `50 33 44 5A`).
2. Reverser l'algorithme via Ghidra sur l'exécutable du jeu original (chercher la routine
   qui consomme le magic `P3DZ` et alloue le buffer décompressé) — probablement un LZ
   maison Radical/Pure3D, distinct du zlib utilisé pour `RZ`.
3. Implémenter le décompresseur dans `src/P3D/`, puis ajouter dans `P3DFile.cpp` une
   branche `if (type == FileTypes::P3DZ)` symétrique à la branche `RZ` (décompresser →
   `_root = std::make_unique<P3DChunk>(uncompressed)`).
4. Valider en comparant la sortie décompressée à un `.p3d` non compressé équivalent connu.

## Critères d'acceptation

- [ ] L'algorithme P3DZ est documenté (format d'en-tête + schéma de décompression).
- [ ] `P3DFile` charge un `.p3d` P3DZ sans déclencher l'`assert(type == P3D)`.
- [ ] La sortie décompressée parse en une arborescence `P3DChunk` valide.
- [ ] Round-trip vérifié : un fichier P3DZ produit la même arborescence que sa version non compressée.
