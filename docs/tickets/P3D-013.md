# P3D-013 — `P3DUtil::GetShaderTexture()` déclaré jamais défini

- **Status:** TODO
- **Priority:** P0
- **Module:** P3D
- **Depends on:** —
- **Files:** `src/P3D/P3DChunk.h:267` (struct `P3DUtil`), `src/P3D/P3DChunk.cpp:23-58` (définitions `P3DUtil`)

## Contexte

La struct `P3DUtil` (`P3DChunk.h`) ne contient en réalité que `GetDrawables` (x2) et
`ConvertColor`. Le ticket annonce une déclaration `GetShaderTexture()` orpheline, mais
`grep -rn GetShaderTexture src/` ne renvoie **aucune occurrence** : ni déclaration, ni
définition, ni appelant. Soit la déclaration a déjà été supprimée, soit elle n'a jamais
existé. À clarifier puis verrouiller pour éviter une régression silencieuse (déclaration
sans `.cpp` = erreur de lien dès le premier appel).

## Approche

1. Confirmer l'état réel : `grep -rn "GetShaderTexture" src/`. Si zéro résultat, le bug
   « déclaré jamais défini » n'existe plus → fermer en documentant.
2. Si une déclaration réapparaît dans `struct P3DUtil` (`P3DChunk.h`) sans corps dans
   `P3DChunk.cpp`, deux options selon le besoin :
   - **Implémenter** : ajouter la définition dans `P3DChunk.cpp`, sur le modèle de
     `P3DUtil::GetDrawables`. Le rôle attendu est de résoudre la texture liée à un
     `Shader` (chunk `ShaderTextureParam`, clé `"TEX"`) → retourner le nom de texture
     (`std::string`) à passer à `ResourceManager`.
   - **Supprimer** : retirer la déclaration si aucun appelant ne la justifie.
3. Décision par défaut : **supprimer** tant qu'aucun consommateur n'existe (YAGNI) ;
   `Render/Shader.cpp` lit déjà les `ShaderTextureParam` directement.

## Critères d'acceptation

- [ ] `grep -rn GetShaderTexture src/` est cohérent : soit 0 occurrence, soit déclaration **et** définition **et** au moins un appelant.
- [ ] Aucune méthode déclarée non définie ne subsiste dans `struct P3DUtil`.
- [ ] Le projet compile et linke sans erreur de symbole non résolu.
