# P3D-004 — Contrôleurs d'animation

- **Status:** TODO
- **Priority:** P2
- **Module:** P3D
- **Depends on:** —
- **Files:** `dev/codegen/p3d.json:965-980` (`FrameController`, `FrameController2`, `MultiController2` **déjà décrits**), `src/P3D/P3DChunk.h:122-139` (ChunkType), `src/P3D/P3D.generated.{h,cpp}`

## Contexte

Chunks de contrôle d'animation, état inégal :

- `FrameController` (0x121200), `FrameController2` (0x121201),
  `MultiController2` (0x121202) : **déjà décrits dans `p3d.json`** (lignes 965-980) mais
  **non régénérés** — et minimalistes (`FrameController` = `version/name/controllerId/numTracks`
  seulement). À compléter puis générer.
- `VertexAnimKeyFrame` (0x121304), `VectorOffsetList` (0x121301),
  `Float1Channel` (0x121100), `Float2Channel` (0x121101),
  `Vector1Channel` (0x121102) : déclarés dans l'enum `ChunkType` mais **absents de
  `p3d.json`** → à reverser. Les canaux existants (`Vector2Channel`, `Vector3Channel`,
  `QuaternionChannel`, `CompressedQuaternionChannel`) servent de modèle (lignes 43-79).

## Approche

1. Compléter les layouts de `FrameController`/`FrameController2`/`MultiController2` dans
   `p3d.json` (reverse des champs manquants : tracks, mapping vers les `Animation`).
2. Décrire les canaux manquants en copiant la structure des canaux existants
   (`param string[4]`, `numFrames u32`, `frames buffer<u16[numFrames]>`,
   `values buffer<...[numFrames]>`, `interpolationMode child ChannelInterpolationMode`) :
   - `Float1Channel` → `values buffer<float[numFrames]>` ;
   - `Float2Channel` → `values buffer<vec2[numFrames]>` ;
   - `Vector1Channel` → `values buffer<float[numFrames]>` (à confirmer).
3. `VertexAnimKeyFrame` + `VectorOffsetList` : reverser (animation de vertices,
   morph targets). `VectorOffsetList` ressemble à `OffsetList`/`ExpressionOffsets`
   (`size u32`, `offsets buffer<vec3[size]>`).
4. Lancer `dev/codegen/run.bat`, vérifier les `Load()`.

## Critères d'acceptation

- [ ] `FrameController`, `FrameController2`, `MultiController2` complétés et générés.
- [ ] `Float1Channel`, `Float2Channel`, `Vector1Channel`, `VertexAnimKeyFrame`, `VectorOffsetList` décrits dans `p3d.json` et générés.
- [ ] Un `.p3d` d'animation utilisant ces canaux/contrôleurs se parse sans assert.
- [ ] Les valeurs de keyframes sont accessibles via getters générés.
