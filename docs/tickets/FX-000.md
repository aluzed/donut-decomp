# FX-000 — Câbler le pipeline FBO post-process (scène → `_sceneFBO` → quad plein écran)

- **Status:** TODO
- **Priority:** P2
- **Module:** Post-FX
- **Depends on:** —
- **Files:** `src/Game.h:116-119`, `src/Game.cpp:183-185`, `src/Game.cpp:833-857`, `assets/shaders/postprocess.vert`, `assets/shaders/postprocess.frag`

## Contexte
`_sceneFBO`, `_postProcessShader`, `_fullscreenQuadVB`, `_fullscreenQuadBinding`
sont **déclarés** (`Game.h:116-119`) et le shader post-process est compilé
(`Game.cpp:183-185`), mais le FBO n'est **jamais bind** : le rendu va directement
au backbuffer (`Game.cpp:833-857`, `glClear` puis draws). Le quad plein écran
n'est pas non plus créé. Aucune passe FX ne peut fonctionner sans cette plomberie.
Prérequis de tous les FX-xxx.

## Approche
1. **Créer le FBO** (constructeur `Game`, près de `Game.cpp:185`) :
   `GL::FrameBuffer::Format` avec `EnableColourBuffer(true)` +
   `EnableDepthBuffer(true)` ; instancier `_sceneFBO` à la taille viewport.
   Le redimensionner sur changement de viewport (`SetSize`, cf. `FrameBuffer.h:86`).
2. **Créer le quad plein écran** : `_fullscreenQuadVB` = 2 triangles en NDC
   (positions xy, le `postprocess.vert` fait `texCoord = position.xy*0.5+0.5`),
   `_fullscreenQuadBinding` avec un `GL::ArrayElement` location 0 (xyz).
3. **Rediriger le rendu scène** : avant le `glClear`/draws (`Game.cpp:833`),
   `_sceneFBO->Bind()` ; rendre le monde + personnage dans le FBO ; puis
   `FrameBuffer::Unbind()`.
4. **Passe finale** : bind backbuffer, `_postProcessShader->Bind()`, binder
   `_sceneFBO->GetColorTexture(0)` sur l'unité 0 + uniform `sceneTex=0` et
   `screenSize`, dessiner le quad. Important : la passe ImGui/HUD
   (`Game.cpp:794-795,821`) doit rester **après** le post-process (sinon le HUD
   est filtré).
5. **Passthrough par défaut** : neutraliser les effets du `postprocess.frag`
   existant (bloom/vignette/grading) via uniforms à 0 / identité, pour valider
   l'égalité d'image avant d'activer les FX.

## Critères d'acceptation
- [ ] La scène est rendue dans `_sceneFBO` puis blittée via le quad plein écran.
- [ ] En mode passthrough (effets neutralisés), l'image est **identique** au rendu
      direct actuel (pas de décalage, pas de flip vertical).
- [ ] Redimensionnement de la fenêtre : le FBO se redimensionne, pas d'artefact.
- [ ] HUD / ImGui restent nets (rendus après la passe post-process).
