# FX-001 — Bloom (flou gaussien 2 passes)

- **Status:** TODO
- **Priority:** P3
- **Module:** Post-FX
- **Depends on:** FX-000
- **Files:** `assets/shaders/postprocess.frag:6,:23-38,:46-50`, `src/Render/OpenGL/FrameBuffer.h`, `src/Game.cpp:183-185`

## Contexte
Le `postprocess.frag` esquisse déjà un bloom **mono-passe** : extraction des zones
brillantes (`bright = max(color - 0.7, 0)`, `:46`), `blurSample` horizontal et
vertical dans le même fragment (`:48-49`, poids gaussiens `:33`) et ajout
(`:50,:57`). C'est un flou séparable mais fait en une passe (qualité/perf
médiocres). Objectif : un vrai bloom 2 passes (ping-pong FBO), après FX-000.

## Approche
1. Créer deux FBO « ping-pong » demi-résolution (`GL::FrameBuffer`,
   `FrameBuffer.h:78`) pour le flou séparable. Un shader de bright-pass
   (seuil `0.7`, cf. `postprocess.frag:46`) écrit dans le premier.
2. Passe horizontale puis verticale alternées en ping-pong (réutiliser
   `blurSample` `postprocess.frag:23-38`, ou shader de blur dédié), N itérations
   pour élargir le halo.
3. Composer : dans la passe finale, additionner le résultat bloom × `bloomIntensity`
   (`postprocess.frag:6`) à la scène, en retirant le bloom mono-passe inline
   actuel (`:46-50`) devenu redondant.
4. Exposer `bloomIntensity` + seuil via uniforms et slider ImGui
   (`Game.cpp` ~l.1277). `bloomIntensity = 0` ⇒ passthrough.

## Critères d'acceptation
- [ ] Les surfaces brillantes diffusent un halo doux et large (>1 passe).
- [ ] `bloomIntensity = 0` redonne une image identique au passthrough.
- [ ] Le bloom utilise des FBO ping-pong séparés, pas le calcul inline mono-passe.
- [ ] Coût maîtrisé (flou en demi-résolution).
