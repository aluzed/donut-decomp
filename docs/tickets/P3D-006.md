## Contexte

Quatre variantes de la hiérarchie d'entités monde. État inégal :

- `InstancedStaticEntity` (0x3F00009) : **déjà présent dans `p3d.json`** (lignes 921-927)
  mais **pas encore régénéré** (absent de `P3D.generated.h`) → il manque juste un
  `run.bat` + branchement.
- `Anim` (0x3F0000C), `AnimDynamicPhysicsWrapper` (0x3F0000F), `AnimCollision` (0x3F00008) :
  déclarés dans l'enum `ChunkType` mais **absents de `p3d.json`** → à reverser.

`AnimDynamicPhysics` (0x3F0000E) et `AnimObjectWrapper` (0x3F00010) servent déjà de modèle
dans `p3d.json` (lignes 225-240).

## Approche

1. **InstancedStaticEntity** : régénérer (`run.bat`), puis ajouter le cas
   `ChunkType::InstancedStaticEntity` dans `Level.cpp` sur le modèle de
   `InstancedStaticPhysics` (`geometry` + `instanceList` → `P3DUtil::GetDrawables`).
2. **AnimDynamicPhysicsWrapper** : décrire dans `p3d.json` en s'inspirant de
   `AnimDynamicPhysics` (probable wrapper : `animDynamicPhysics: child AnimDynamicPhysics`
   + `frameController`/`compositeDrawable`). Reverser via Ghidra ou dumps.
3. **Anim** / **AnimCollision** : reverser les layouts ; `AnimCollision` agrège
   vraisemblablement une `Animation` + un `CollisionObject` (cf. `p3d.json` existant).
4. Régénérer, vérifier les `Load()`, brancher les nouveaux cas dans `Level.cpp`.
